# layout-composer

Class: derived.
Use rule: advisory module taxonomy only.

## Purpose

Create or restructure page and screen layouts.

## Allowed Modes

- generate
- refine
- normalize

## Required Inputs

- artifact description
- page goal
- content blocks
- product type
- device targets

## Outputs

- layout blueprint
- region map
- proportion rationale
- responsive notes

## Must Not Do

- Change semantic product logic.
- Invent missing content.
- Treat dashboards like editorial pages.
