# Onboarding Router

Load context in layers.

## Layer 1 - Base Context
- What kind of system this is
- Global epistemic and governance rules
- Expected response discipline

## Layer 2 - Domain Context
- Repo / migration / ops / helpdesk / research
- Source-of-truth ordering
- Domain boundaries

## Layer 3 - Mode Context
- Analysis / review / planning / migration / runtime / helpdesk / implementation

## Layer 4 - Task Context
- Goal
- Scope
- Inputs
- Output format
- Explicit non-goals

## Routing rule
Do not begin execution until the required context layer is active.
